# 明日の駅 (asu-no-eki)

> 1 時間電車通勤する会社員に、 毎朝 1 駅だけ「明日この駅でこんなことが起こる」 を届ける。

通勤の窓越しに見える駅が、 5 年後にどう変わるのか。 来週どこの工事が始まるのか。 どのビルが OPEN するのか。 1 駅 1 件、 60 字以内。 毎日 1 駅だけ自動でハイライトされる。 そのほかの 29 駅は地味なリスト、 知りたければクリック。

## 何ができる

- 山手線 30 駅の都市計画 / 再開発 / 駅周辺の変化を 1 駅 3-4 件、 1 行ずつまとめた静的データセット
- 「今日の一駅」 は日付 (YYYY-MM-DD) を seed にして deterministic に選ぶ — 同じ日に開けば同じ駅、 違う日は別の駅
- 山手線環状を SVG で描いた dashboard、 今日の駅は 鈍金で大きく、 他の駅は灰色の点
- FastAPI で 4 つの API エンドポイント (/api/stations, /api/stations/{id}, /api/today, /)

## 哲学

- 通勤は「ただ移動するだけの時間」 にしない、 毎朝 1 駅だけ「ここに何が起きるか」 を知っておく
- 駅を「変わる場所」 として見直すための小さな窓
- 「明日この駅は閉鎖!」 のような押し付けはしない、 単に事実を 1 行
- streak も「今日も訪問」 のような毎日訪問プレッシャーも出さない

## 走らせる

```bash
cd asu-no-eki
pip install -e ".[test]"
pytest                          # 25+ ユニット + API テスト
asu-no-eki                      # uvicorn で起動
# http://localhost:8000/
```
