# takibi 実装計画

## Phase 1: プロジェクト基盤
- index.html (エントリーポイント、メタタグ、PWA manifest リンク)
- style.css (ベーススタイル、ダークテーマ、レスポンシブ)
- app.js (メインロジック、タイマー、状態管理)
- manifest.json (PWA設定)
- service-worker.js (オフラインキャッシュ)

## Phase 2: 焚火アニメーション
- canvas/fire.js: ピクセルアートの焚火を描画
  - 炎の揺らぎ（複数レイヤーのノイズベース）
  - 舞う火の粉（パーティクル）
  - 薪の描画
  - 火の大きさ変化（focus サイクル完了で成長）

## Phase 3: 環境音
- audio/ambience.js: Web Audio API で生成
  - パチパチ音（ホワイトノイズ + バンドパス + エンベロープ）
  - 風の音（ローパスフィルタを通したノイズ）
  - 音量フェード（ON/OFF）

## Phase 4: タイマー & UI
- 25分集中 → 5分休憩サイクル
- 静かな視覚通知（火の挙動が変わる）
- 「火を灯す / 消す」ボタン
- 「音を聞く / 静かにする」ボタン
- 集中サイクル数を画面下部に静かに表示

## Phase 5: PWA & オフライン
- service-worker でキャッシュ
- manifest.json でホーム画面アイコン
- スマホ縦向き最適化

## Phase 6: テスト
- tests/ ディレクトリにユニットテスト
- タイマーロジック、状態遷移、サイクル完了判定
- ブラウザ実行可能なテストランナー

## 品質ゲート
- iOS Safari, Android Chrome での描画確認
- 375px 幅で崩れない
- 音声がモバイルブラウザで再生できる（ユーザー操作後に開始）
- 全UIテキストが日本語
- ピクセル感のあるシャープな描画（image-rendering: pixelated）
