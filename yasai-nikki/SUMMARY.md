# やさい日記 (yasai-nikki) — SUMMARY

## 何を作ったか

6 〜 12 歳の子どもが、1 日 1 分だけ開いて野菜の観察記録をつけていく Raylib のピクセルアートアプリです。28 日間続けると、毎日の「葉の数 / 色 / きょうのきもち」が並んだ観察カレンダーが完成。10 年後に大人になったその子が開いても「あの夏、トマトをこんなふうに見ていたんだ」と読み返せる日記が残ります。

「記録して残す」(Intent 6) を狙ったため、ゲーム性ではなく **データの永続性** に重きを置きました。ユーザーの観察データは絶対に破棄しない方針。

## 技術スタック

- **C99 + Raylib 5.5** (Homebrew installed)
- **ロジック分離**: `crop.c` `diary.c` は Raylib 完全不依存。テストは別バイナリでビルド
- **永続化**: ホームディレクトリ `~/.yasai-nikki.json` (手書き JSON シリアライザ + ad-hoc パーサ)
- **日本語フォント**: macOS のヒラギノ角ゴシックを LoadFontEx で読み込み
- **ピクセルアート**: 矩形・楕円・三角の組合せで 5 種 × 6 段階を描画

## 技術的判断

| 課題 | 解決 |
|---|---|
| 子ども向けの語彙レベル | ひらがな多め、小学校 4 年生までの漢字に制限 (UI / プロンプトのみ) |
| キーボード入力ゼロ | 全選択を ← → と Space / Enter で完結。タイピング不要 |
| 観察データを失わない | アプリ起動時にディスクから自動ロード、各 commit と ESC で自動保存 |
| 短期間でも気軽に続けられる | 1 日 3 つの観察カード (葉数 / 色 / 気分) のみ、選択肢式 |
| JSON ライブラリ依存を避ける | ad-hoc に小さな JSON パーサを実装 (key 検索＋数値配列読み) |
| Raylib 非依存テストを書く | `crop.c` `diary.c` を `test_diary.c` から直リンクして純粋関数テスト |

## ファイル構成

```
yasai-nikki/
├── Makefile             pkg-config raylib
├── src/
│   ├── crop.{h,c}      5 種 × 6 段階の野菜定義
│   ├── diary.{h,c}     状態機械 + JSON シリアライズ
│   ├── render.c        Raylib ピクセルアート描画
│   └── main.c          入力 + 永続化
└── tests/
    └── test_diary.c    純粋ロジックテスト (101 件)
```

## 動かし方

```bash
brew install raylib
cd yasai-nikki
make
./yasai-nikki

make test                # 101 件のユニットテスト
```

## 数値

- **テスト**: 101 / 101 passing
- **コード**: C 約 850 行 (src/ + tests/)
- **バイナリ**: 53KB arm64 release
- **画面**: 1280×720, 60 FPS

## 次のステップ案

- 音 — 観察決定時の優しいチャイム、ステージ進化時の祝福音
- 親モード — 大人が子どもの観察データを PDF/印刷できる「観察日記 印刷」
- 季節を超えてアーカイブ — 完成した 1 シーズンを別ファイルに「卒業」させ、新シーズンを始める
- 観察カード追加 — 「におい」「さわった感じ」「太陽はあったか」など五感系
- 親子モード — 親と子がそれぞれの観察を書き、後で並べて比較
