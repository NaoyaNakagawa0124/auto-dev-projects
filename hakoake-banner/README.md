# 箱開けバナー (hakoake-banner)

> Hacker News を 開くたび、 あなたの 架空 の 引っ越し 進捗 が 出る Chrome 拡張機能。

「あなたは 引っ越し 137 日目 です。 ダンボール 箱 は -32 個 です。」

— と、 完全に 根拠 ゼロ の 「引っ越し 進捗」 が、 HN の ページ 上部 に、 そっと 表示される。 箱 の 数 は 負 にも なる。 訪問回数 が 多いほど マイナス が 深くなる。 引っ越し 直後 の 人 を 想定 した バナー だが、 HN を 見るのは ほぼ 開発者 で、 引っ越しとは 関係ない。 そこ が 笑い。

## 何のためにある?

- 引っ越し 直後 の 人 は HN を 見ない (むしろ 引っ越し に 忙しい)
- HN を 見るのは ほぼ 開発者 で、 引っ越し 中で ない こと が 多い
- なのに、 拡張機能 は 全員 を 「引っ越し 直後 の 人」 として 扱う
- 完全 な 非接続 が 笑い を 生む

ターゲット は コメント 欄 で 「いつまで HN 見てるの」 と 友達 に 笑われる、 そんな ユーザー。

## 仕組み

- 初回 インストール 時刻 を localStorage に 記録 (= 引っ越し 開始日)
- HN を 訪問 する たびに 「箱 を 1 個 だけ 開ける」 という 偽 の 進捗 を 進める
- 経過日数 ÷ 訪問回数 で 「箱 残り」 を 計算、 訪問 が 多すぎると 負 に なる
- 50 種類 の バナー フレーズ から、 訪問時刻 を seed として 1 件 を 表示

## 走らせる

```bash
cd hakoake-banner
npm install
npm test                # ロジック ユニット テスト (Chrome 不要)
```

Chrome / Edge / Brave で `chrome://extensions` を 開き、 「デベロッパー モード」 を ON、 「パッケージ化されていない拡張機能を読み込む」 で `hakoake-banner/` フォルダ を 選択。

その 後、 `https://news.ycombinator.com/` を 開くと バナー が 出る。
