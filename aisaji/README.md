# 相匙 (aisaji)

> 「夕飯どうする?」 を 90 秒で。 2 人の匙加減で。

ストレスを抱えた管理職が、 帰宅した瞬間に「今夜なに食べる?」 で 30 分会議するのを止めるための、 90 秒だけ動く小さな Web ゲーム。

## どう使うか

1. 1 台のスマホを 2 人で持つ (1 人が左半分、 1 人が右半分にそれぞれ親指を置く)
2. 「始める」 を押す
3. 90 秒のあいだ、 料理カードが 1 枚ずつ 6 秒くらいで切り替わる
4. それぞれが「気が向く」 と思ったら自分の側を **同時に** タップする
5. 90 秒経つと、 両方が「気が向く」 した料理から 1 枚だけランダムに「今夜の夕飯」 として表示される
6. 両方一致しなかった場合は「今夜は外食でいい」 が出る

合計タップ数も、 streak も、 連勝記録も出さない。 90 秒経ったらアプリを閉じる。

## 哲学

- 「今夜の食事を決める」 のはストレス管理職にとって本気で重い意思決定。 アプリの仕事はその意思決定を 30 分の議論から 90 秒のゲームに置き換えること
- カードは「カレー」 「親子丼」 のような普通の家庭料理 + 「外食」 「冷凍餃子」 「ピザ宅配」 「茶漬けでいいか」 のエスケープカードを含む
- 両方が同じカードを選んだ瞬間に「決まり」 が成立、 後で覆せない
- 終わったら「お疲れさま、 今夜は鯖の塩焼き」 のような短い一行で締める

## 技術

- Vanilla HTML / CSS / ES Module
- 純 JS ロジック (Vitest でテスト可能)
- xorshift32 シードランダム (`pickHand(seed, n)` でカード選びが決定的)
- localStorage で 7 日分の履歴 (見返したくない人は無視できる)
- 単一 HTML、 ビルド不要、 `python3 -m http.server` か Vercel/Netlify Static で即配信可能

## 走らせる

```bash
cd aisaji
npm install
npm test                       # Vitest で 28 ユニットテスト

# 配信 — プロジェクトルート から起動する (web/app.js が ../src を import するため)
python3 -m http.server 8080
# http://localhost:8080/web/
```
